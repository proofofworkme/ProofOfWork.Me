# Simple Design System Skill (Codex)

## Mission
Create practical Simple design UI with clean typography, neutral colors, intuitive layouts, explicit states, and accessible interactions.

## Style Foundations
- Visual style: minimal, clean.
- Typography scale: 12/14/16/20/24/32.
- Fonts: primary=Inter, display=Inter, mono=JetBrains Mono.
- Weights: 100, 200, 300, 400, 500, 600, 700, 800, 900.
- Color palette: primary, neutral, success, warning, danger.
- Tokens: primary=#3B82F6, secondary=#8B5CF6, success=#16A34A, warning=#D97706, danger=#DC2626, surface=#FFFFFF, text=#111827.
- Spacing scale: 4/8/12/16/24/32.

## Accessibility
WCAG 2.2 AA, keyboard-first interactions, visible focus states.

## Writing Tone
Concise, confident, helpful.

## Rules: Do
- Prefer semantic tokens over raw values.
- Preserve visual hierarchy.
- Keep interaction states explicit.

## Rules: Don't
- Avoid low contrast text.
- Avoid inconsistent spacing rhythm.
- Avoid ambiguous labels.

## Quality Gates
- Prioritize accessibility and clarity over novelty.
- Anchor visual choices to tokens, thresholds, or examples.
- Prefer system consistency over one-off local optimizations.
- Include responsive behavior for long labels, empty states, and overflow.
